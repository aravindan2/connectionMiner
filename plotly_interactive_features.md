# Plotly Interactive Features for Multi-Panel UMAP Visualizations

This document documents the interactive Plotly.js features implemented in `viz_combined_three_panel.html` (generated by `cm_visual/viz_plotly.py::_viz_combined_three_panel` and `_inject_cross_highlight_js`), so future agents can replicate or extend them in other visualizations.

---

## Feature 1: Cross-Panel Highlighting on Hover

**File:** `cm_visual/viz_plotly.py` — `_inject_cross_highlight_js()` (JS block)

**Behavior:** Hovering over a point in any of the three UMAP panels highlights the same cell in all three panels with a red circle marker.

**Implementation pattern:**

1. **Overlay highlight traces:** After the 3 main data traces, add 3 "highlight" traces (one per subplot) with `x=[None], y=[None]` so they render empty initially:

   ```python
   for _col in [1, 2, 3]:
       fig.add_trace(
           go.Scattergl(
               x=[None], y=[None],
               mode="markers",
               marker=dict(size=14, color="rgba(255, 50, 50, 0.9)",
                           line=dict(width=2, color="white"), symbol="circle-open"),
               showlegend=False, hoverinfo="none", name="highlight",
           ), row=1, col=_col,
       )
   ```

2. **On `plotly_hover`:** Use `Plotly.restyle` to set the highlight trace's `x` and `y` to the hovered point's coordinates from each main trace:

   ```javascript
   plotDiv.on('plotly_hover', function(data) {
       var pt = data.points[0];
       var idx = pt.pointIndex;
       var hl = hlIdx();  // [n-3, n-2, n-1] — last 3 traces are highlights
       for (var c = 0; c < 3; c++) {
           var x = plotDiv.data[mainIdx[c]].x[idx];
           var y = plotDiv.data[mainIdx[c]].y[idx];
           Plotly.restyle(plotDiv, { 'x': [[x]], 'y': [[y]] }, hl[c]);
       }
   });
   ```

   Trace indices: main traces are always `[0, 1, 2]` (added first), highlight traces are always the last 3.

3. **On `plotly_unhover`:** Reset highlight traces back to `null`:

   ```javascript
   plotDiv.on('plotly_unhover', function() {
       var hl = hlIdx();
       for (var c = 0; c < 3; c++) {
           Plotly.restyle(plotDiv, { 'x': [[null]], 'y': [[null]] }, hl[c]);
       }
   });
   ```

**Performance note:** Updates only 3 × 1 data points per hover instead of restyling all points.

## Feature 2: Consolidated Info Panel (with Panel-Specific Context)

**File:** `cm_visual/viz_plotly.py` — `_inject_cross_highlight_js()` (CSS + HTML + JS)

**Behavior:** A fixed-position side panel on the right shows consolidated cell information organized into sections. The header bar changes color and label depending on which panel was hovered.

### 2a. Python data pipeline — `customdata` with 15 fields

Build a `customdata` NumPy array with every available metadata field per cell, column-stacked as strings:

```python
customdata = np.column_stack([
    barcodes,                    # 0: BARCODE
    annotations,                 # 1: ANNOTATION
    tiers,                       # 2: TIER
    cell_to_meta.astype(str),    # 3: META_ID
    [type_names[t] ...],         # 4: INF_TYPE
    cell_P_max.astype(str),      # 5: CONFIDENCE
    cell_entropy_norm.astype(str), # 6: ENTROPY
    meta_sizes[...].astype(str), # 7: META_SIZE
    clusters_raw,                # 8: CLUSTER (MultiomeNN)
    n_allowed.astype(str),       # 9: N_ALLOWED (# constraint types)
    cell_family,                 # 10: FAMILY
    cell_subsystem,              # 11: SUBSYSTEM
    cell_category,               # 12: CATEGORY
    umap_x,                      # 13: UMAP_X
    umap_y,                      # 14: UMAP_Y
])
```

All values must be strings (Plotly serializes them to JSON). The `customdata` is attached identically to all three main traces.

### 2b. HTML structure

```html
<div id="cell-info-panel">
    <div id="cell-info-header">Cell Information</div>
    <div id="cell-info-body">
        <div id="cell-info-content">
            <div class="no-selection">Hover over a cell to see details</div>
        </div>
    </div>
</div>
```

- Outer panel: `position: fixed; top: 60px; right: 10px; width: 280px;` with card styling
- Header bar: colored background that changes per panel
- Content: dynamically populated by JavaScript

### 2c. JavaScript — `renderInfo` with sections

The `panelMeta` array maps `curveNumber` (0, 1, 2) to panel metadata:


The five organized sections render into `#cell-info-content`:

| Section | Fields |
|---------|--------|

| **Type Annotation** | Annotated Type, Family, Subsystem, Category |

Panel-specific key info at the top (highlighted box):

On unhover, the header resets to `"Cell Information"` (blue) and content to placeholder text.

## Feature 3: Synchronized Zoom/Pan Across Subplots

**File:** `cm_visual/viz_plotly.py` — `_inject_cross_highlight_js()` (JS block, `plotly_relayout` handler)

**Behavior:** Zooming or panning on any of the three subplot panels automatically applies the identical zoom/pan to the others.

**Implementation pattern:**

1. **Axis naming in Plotly subplots:**
   - Subplot 1 (row=1, col=1): `xaxis`, `yaxis`
   - Subplot 2 (row=1, col=2): `xaxis2`, `yaxis2`
   - Subplot 3 (row=1, col=3): `xaxis3`, `yaxis3`

2. **Listen for `plotly_relayout`:** Fires on zoom, pan, or reset axes.

3. **Detect which axis changed** (handles both Plotly event formats):

   ```javascript
   function getRange(ed, key) {
       if (ed[key] && Array.isArray(ed[key].range) && ed[key].range.length === 2)
           return ed[key].range;
       var r0 = ed[key + '.range[0]'], r1 = ed[key + '.range[1]'];
       if (r0 !== undefined && r1 !== undefined) return [r0, r1];
       return null;
   }
   ```

4. **Apply to sibling axes:** When `xaxis` range changes → apply to `xaxis2`, `xaxis3`. Same for `yaxis`.

5. **Guard against infinite recursion:**

   ```javascript
   var _syncing = false;
   plotDiv.on('plotly_relayout', function(ed) {
       if (_syncing) return;
       if (!ed || Object.keys(ed).length === 0) return;  // skip init events
       // ... build update ...
       if (Object.keys(update).length > 0) {
           _syncing = true;
           Plotly.relayout(plotDiv, update);
           setTimeout(function() { _syncing = false; }, 200);
       }
   });
   ```

## General Pattern: JS Injection into Plotly HTML

**File:** `cm_visual/viz_plotly.py` — `_inject_cross_highlight_js()`

Plotly's `fig.write_html()` generates a self-contained HTML page. To add custom behavior:

1. **Generate HTML with known `div_id`:**
   ```python
   fig.write_html(html_path, include_plotlyjs="cdn", div_id="three-panel")
   ```

2. **Read back and inject before `</body>`:**
   ```python
   with open(html_path, "r") as f:
       html = f.read()
   html = html.replace("</body>", css_block + "\n" + info_panel_html + "\n" + js_block + "\n</body>")
   with open(html_path, "w") as f:
       f.write(html)
   ```

**Why:** `fig.write_html()` handles Plotly.js loading and trace embedding. Post-hoc injection lets you add arbitrary JS event handlers and DOM elements without forking Plotly's HTML generation.

## Summary of Plotly Events Used

| Event | Purpose | Handler Signature |
|-------|---------|-------------------|
| `plotly_hover` | Detect hover + identify cell | `function(data) { data.points[0].pointIndex, data.points[0].customdata, data.points[0].curveNumber }` |
| `plotly_unhover` | Reset highlights + info panel | `function(data)` |
| `plotly_relayout` | Detect zoom/pan changes | `function(eventData) { eventData['xaxis.range[0]'], eventData['xaxis.range[1]'] }` |

## Reusable Snippet: `customdata` for Scattergl Traces

```python
fig.add_trace(go.Scattergl(
    ...,  # x, y, mode, marker
    customdata=np.column_stack([field1, field2, ...]),
    hovertemplate=(
        "<b>%{customdata[0]}</b><br>"
        "Field 2: %{customdata[1]}<br>"
        "<extra></extra>"
    ),
))
```

The `<extra></extra>` removes the default trace name from Plotly's native tooltip. All customdata values must be strings.

## Architecture Summary

```
_viz_combined_three_panel()          # builds figure + customdata
    └── fig.write_html()             # generates base HTML
    └── _inject_cross_highlight_js() # post-hoc injection
            ├── CSS block            # info panel styling
            ├── HTML block           # info panel DOM elements
            └── JS block             # event handlers + renderInfo
                    ├── plotly_hover    → cross-highlight + renderInfo
                    ├── plotly_unhover  → reset highlights + panel
                    └── plotly_relayout → zoom/pan sync
```
